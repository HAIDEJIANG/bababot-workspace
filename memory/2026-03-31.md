## ⚠️ 重要教训 (01:55)

**错误类型：** 误用 write 覆盖 memory 文件（连续多次）

**用户提醒：** 请吸取教训，不要再犯这样的错误

**规则固化：**
- ✅ Memory 文件操作 → 必须先 read 现有内容，再追加新内容
- ❌ 绝不可直接用 write 覆盖已有 memory 文件
- ✅ 每次操作前检查文件是否存在
- ✅ 存在 → read + append（合并后写入）；不存在 → write 新建

**根本原因：** write 工具是覆盖式写入，必须先获取原内容再合并

此教训必须深刻记住，写入 AGENTS.md 作为长期行为准则。
