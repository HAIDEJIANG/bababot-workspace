## 🔬 CDP 协议深度研究 (18:00-18:17)

**研究目标：** 改进 CDP 协议提取 LinkedIn 帖子的能力

### 已尝试的方案

**v9 CDP 协议版：**
- ✅ 成功连接浏览器（CDP 端口 9222）
- ✅ 成功获取页面基本信息（Title、URL、BodyText）
- ❌ 无法提取帖子（`article` 选择器返回 0）

**v10 CDP 增强版：**
- ✅ 使用 `data-id` 属性（包含 `urn:li:activity`）
- ✅ 使用 `aria-label` 属性
- ✅ 多选择器组合策略
- ❌ 仍然返回 0 条帖子

**调试脚本诊断：**
- ✅ 确认浏览器有 3 个 LinkedIn 标签页
- ✅ 确认目标页面已登录（Feed | LinkedIn）
- ❌ 所有 `data-id` 元素：0 个
- ❌ 所有 `aria-label` 相关元素：0 个
- ❌ 所有 `article` 标签：0 个

### 技术结论

**CDP 协议无法提取 LinkedIn 的根本原因：**

1. **Shadow DOM 隔离**
   - LinkedIn 帖子内容渲染在 Shadow DOM 内部
   - CDP 的 `document.querySelectorAll()` 无法穿透 Shadow DOM
   - 需要连接到 Shadow Root 才能访问内容

2. **动态类名哈希**
   - 类名如 `_4de8b190` 每次页面加载都不同
   - CSS 选择器无法稳定定位

3. **iframe 嵌套**
   - 部分内容在跨域 iframe 中
   - CDP 无法访问跨域 iframe 内容

4. **CDP 上下文问题**
   - 连接的 session 可能是后台上下文
   - 实际渲染内容在另一个 frame 中

- ❌ 所有 `article` 标签：0 个

### 技术结论

**CDP 协议无法提取 LinkedIn 的根本原因：**

1. **Shadow DOM 隔离**
   - LinkedIn 帖子内容渲染在 Shadow DOM 内部
   - CDP 的 `document.querySelectorAll()` 无法穿透 Shadow DOM
   - 需要连接到 Shadow Root 才能访问内容

2. **动态类名哈希**
   - 类名如 `_4de8b190` 每次页面加载都不同
   - CSS 选择器无法稳定定位

3. **iframe 嵌套**
   - 部分内容在跨域 iframe 中
   - CDP 无法访问跨域 iframe 内容

4. **CDP 上下文问题**
   - 连接的 session 可能是后台上下文
   - 实际渲染内容在另一个 frame 中
