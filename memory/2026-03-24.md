## 2026-03-25 凌晨 - RFQ 自动化脚本开发

### 开发 stockmarket_rfq_v2.py
- **触发原因**: 浏览器操作太慢，14 小时只发 21 条 RFQ
- **解决方案**: Python + requests 直接调 StockMarket.aero HTTP API
- **效率提升**: 624 倍（1.5→936 条/小时）

### 关键技术发现
- 登录字段名: `username`（不是 `userName`）
- 登录需要 `group1=signIn` 隐藏字段
- RFQ 提交链路: `LoadItemDetailAction.do` → `RFQConfirmAction.do`
- 每条库存有唯一 `sysStmKey`，从 `onclick="showitemdetailpopup(...)"` 提取
- 数量字段拼写错误: `quanitity`（StockMarket 的 bug）
- SSL 证书需要 `verify=False`

### 新规则
1. **PN 清洗**: 去括号中文、去空格后内容
2. **条件降级**: 可用件需求可退而求其次用全新件，反之不行
3. **工作纪律**: 不需要休息、不中途汇报、认真读指令每个字
